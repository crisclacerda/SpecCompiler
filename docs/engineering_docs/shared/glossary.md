## Glossary

| Term | Definition |
|------|------------|
| **[TERM-AST](@)** | Abstract Syntax Tree - Pandoc's internal document representation |
| **[TERM-EAV](@)** | Entity-Attribute-Value - Database pattern for flexible attributes |
| **[TERM-04](@)** | Numbered element (figure, table, diagram) that can be cross-referenced |
| **[TERM-16](@)** | Pipeline component that processes documents in a specific phase |
| **[TERM-HLR](@)** | High-Level Requirement |
| **[TERM-PID](@)** | Project Identifier - Unique ID for objects (e.g., `@REQ-001`) |
| **[TERM-PROOF](@)** | SQL query that validates data integrity |
| **[TERM-IR](@)** | Specification Intermediate Representation - SQLite database schema |
| **[DIC-SPEC-001](@)** | Root document container (one per .md file) |
| **[DIC-OBJ-001](@)** | Individual item within a specification (requirement, test, section) |
| **[TERM-TYPE](@)** | Category definition for objects, floats, relations, or views |
| **[TERM-VC](@)** | Verification Case |
| **[DIC-VW-001](@)** | Generated content block (TOC, LOF, abbreviation list) |
